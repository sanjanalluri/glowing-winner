GaelShare: Arranging carpools for DHS students since 2021

Capable of easily arranging carpools for DHS students, GaelShare is presented in a website format and is coded completely in HTML/CSS. Bootstrap, an open-source CSS framework, is also used to more effectively structure a responsive webpage. The website currently harbors three pages, two of which serve as forms to collect user input. The data received from these forms can then be processed through a web server interface and fed into a back-end algorithm. Our website's front-end, focused on effectiveness and simplicity, can smoothly receive user input and presents information using a straightforward, clean, and readable layout.

GaelShare offers a:

  • Responsive, accessible, and easy to navigate webpage

  • Reliable backend algorithm and effective user-input features

  • Rideshare website targeted towards the needs of DHS students

Powered by a Python Algorithm

In order to find rides for DHS students, a Python backend algorithm will be used to pair students up with their riders, or other parents from the DHS community. In this Python algorithm there are currently three main components, as shown here. There are three functions, the first called close_distance(), which determines if the potential rider and driver are close enough to be considered to carpool together. To determine this information given the address of both users, a Python package called geopy is used to provide exact locations. This code only considers a rider and driver available to carpool as long they are within 1 mile of each other. The second function close_time() only considers a rider and driver compatible if their preferred pick-up times are within 15 minutes of each other. The last function carpool() calls the previous two functions to make a final decision; if both functions agree that the pair are suitable to carpool, it returns True, and False otherwise. This code uses sample data to replace user data that would otherwise be obtained through forms on the website, as well as example runs to see how the program works given multiple different cases.



Data Pathways Modeled Using NetLogo

GaelShare involves two programming languages: HTML/CSS and Python. It is important to understand how the information inputted into the HTML/CSS website is transferred to the Python algorithm so that compatible riders can be matched together. In order to visualize the communication between the two languages, we used Netlogo to create a simulation. Netlogo is a programming software that allows the user to experiment with the creation of different environments and observe the behavior of agents that interact within that environment.

The yellow patch represents the HTML/CSS website. The red patch represents the Python algorithm. The agent, or the car, represents the user-inputted information traveling from side to side. The road represents the WSGI (Web Server Gateway Interface), which connects the Python to the HTML/CSS. The car's original color is yellow when it is in the yellow patch (the HTML/CSS website). However, as it crosses the road into the red patch (the Python algorithm), it turns red. This is the Python algorithm interpreting the inputted information and matching riders together. The car then travels back to the HTML/CSS website, notifying riders of their successful match.

Depicting the communication of HTML/CSS and Python in such a way allows inexperienced users to understand the way our website functions.
